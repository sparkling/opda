## 7. G1 code-fix — `term_sourcing.py` updates

The G1 deferred-work follow-up (per ADR-0005 §G) had its
documentation closed but the code half pending. Resolved this round:

**Before (pre-G1):**

```python
# slot ordering: W3C → TF → glossary → dictionary → regulator
if term_id in W3C_REGISTRY: ... tier=1
elif term_id in OPDA_TF_REGISTRY: ... tier=2
elif glossary and term_id in glossary: ... tier=3
elif dictionary and term_id in dictionary: ... tier=4
elif term_id in EXTERNAL_REGULATORS: ... tier=5  # WRONG: should be slot 3
```

**After (G1-corrected):**

```python
# slot ordering matches ODR-0004 §7a verbatim:
# slot 1 = W3C; slot 2 = TF; slot 3 = regulator (CONTEXTUAL);
# slot 4 = glossary; slot 5 = dictionary
def resolve_term(...) -> ResolvedTerm:
    primary = _primary_lookup(...)     # slots 1, 2, 4, 5 (skips slot 3)
    contextual = _contextual_lookup(...)  # slot 3 (always evaluated)
    return ResolvedTerm(term_id, primary, contextual)
```

`ResolvedTerm` dataclass:

```python
@dataclass(frozen=True)
class ResolvedTerm:
    term_id: str
    primary: SourceRecord | None      # from slot 1/2/4/5
    contextual: list[SourceRecord]    # from slot 3 (zero or more)
```

Test updates: `test_term_sourcing.py` grew from 7 → 10 tests covering:

- `test_slot1_w3c` (renamed from `test_tier1_w3c`) — verifies `primary`
- `test_slot2_opda_tf` — verifies `primary` from slot 2
- `test_slot3_regulator_only_is_contextual` (NEW) — verifies that a
  regulator-only term resolves with `primary=None` and a non-empty
  `contextual` list (previously raised `UnsourceableTerm`)
- `test_slot4_glossary` (renamed from `test_tier3_glossary`)
- `test_slot5_dictionary` (renamed from `test_tier4_dictionary`)
- `test_unsourceable_raises`
- `test_precedence_w3c_beats_glossary`
- `test_contextual_always_evaluated_alongside_primary` (NEW) — verifies
  the helper functions confirm contextual is always evaluated even
  when primary resolved
- `test_external_regulators_registry_export` (NEW) — verifies the
  registry is importable as part of the public API
- `test_source_record_dataclass_fields` (NEW) — verifies `SourceRecord`

The legacy `tier=5 contextual` semantics are gone; slot 3 is now the
documented contextual slot, matching ODR-0004 §7a Knublauch demand.
Downstream emitters consuming `resolve_term` will receive
`ResolvedTerm.primary.source_url` for `dct:source` and zero or more
`ResolvedTerm.contextual[i].source_url` for `skos:scopeNote` /
`skos:closeMatch`. (Currently no emitter consumes `resolve_term`
beyond the test suite — the vocabularies + classes emitters
hand-construct their `dct:source` URIs from ODR sections directly,
which is correct because the term_sourcing module's purpose is to

## 9. Surfaced ambiguity (route engineering vs Council)

**Engineering-resolved (this ADR's scope):**

1. **Foundation `owl:versionIRI` vs `__version__` decoupling.** The
   class-graph IRI stays at 0.3.0 (no class declarations added);
   `__version__` bumps to 0.4.0 (substantive substrate addition). This
   is an engineering choice consistent with the existing pattern
   (ADR-0010 added SKOS at its own version-IRI without bumping
   foundation). No ODR ambiguity.
2. **`PIIWithoutDPVCoAnnotationRule` severity.** ADR-0012 §"SHACL-AF
   rule emission" explicitly specifies `sh:Warning` rather than the
   default `sh:Info` per ODR-0017 §1a. Engineering followed the
   ADR-cited override; no Council issue.
3. **Cat 4 special-category PII shape — predicate name.** The shape
   targets `opda:Person` with a placeholder `opda:hasSpecialCategoryData`
   datatype property. The SKOS `SpecialCategoryScheme` member enumeration
   is deferred per ODR-0011 §Operational specifications ("no S012 Q3
   enum currently scoped"); the shape emits as a structural
   placeholder that activates when the scheme members materialise.
   No Council issue — this is downstream of ADR-0011's deferral.
4. **Per-module shapes for descriptive Kinds — minimum coverage.**
   Each of the five descriptive Kinds (Survey, EPCCertificate, Search,
   Valuation, Comparable) emits a `<Kind>IdentityKeyShape` targeting
   `prov:wasGeneratedBy` (the PROV-O subclass relationship). This is
   the minimum needed to fulfill Cat 1 coverage; richer per-attribute
   shapes (built form, energy rating, etc.) await ADR-0013 overlay
   profiles. No Council issue.
5. **G1 code-fix landing here vs separate ADR.** The brief explicitly
   asked for the G1 fix to land in this commit. No Council issue.

**No ambiguity routed to Council.** All decisions land cleanly within
the cited ODR `## Rules`.
