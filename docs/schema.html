Schema

The physical model of the PDTF schema codebase — the actual repositories, files, packages and engines that turn the data standard into something software can validate against. Where Modelling covers the semantic layer — glossaries, schemes, the ontology — this section covers the concrete artefacts that layer is built on.

Two repositories
  • @pdtf/schemas v3.5.0 — the base transaction schema, 18 form overlays, 16 extension overlays, the verified-claims wrapper, and the merge & validation engine.
  • @pdtf/trust-framework v0.3.0 — the W3C Verifiable Credential schema, JSON-LD contexts and DID-based registry for PDTF v2.
