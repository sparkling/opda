# SKOS substrate

23 `skos:ConceptScheme` instances + 137 `skos:Concept` members emitted by `opda-gen` into `opda-vocabularies.ttl` per ADR-0010. Each scheme is classified into the 7-category UFO framework per ODR-0011 §1a + §8a.

## Source

[`opda-vocabularies.ttl`](../../../../source/03-standards/ontology/opda-vocabularies.ttl) — 1165 lines, generated by `opda-gen` 1.0.0.

## 7-category UFO framework

Per ODR-0011 §8a, every scheme carries `opda:ufoCategory` from one of seven UFO categories. The framework distinguishes ontologically-distinct kinds of enumeration:

| Category | Description | Schemes |
|---|---|---|
| Substance Kind label | Sortal labels naming a substantial Kind | OwnerType, PropertyType, TenureKind |
| Quale-in-Region | Quality particulars over a quality region (DOLCE) | BuiltForm, CentralHeatingFuelType, CouncilTaxBand (EW + Scotland), CurrentEnergyRating, HeatingType, OffMainsDrainageSystemType, OwnershipType, YesNo, YesNoNotApplicable, YesNoNotKnown, YesNoNotRequired |
| Quality Value | DOLCE Quality Region values | AddressVariant, AssuranceLevel, EvidenceMethod |
| Method/plan code | Action-modelling codes (Guizzardi & Wagner 2010) | MilestoneKind, SellersCapacity |
| Phase label | Intra-Kind phase labels | ParticipantStatus, TransactionStatus |
| Role label | Anti-rigid Role labels | Role |
| (Trust-tier label) | (reserved for future use) | — |

The 7-category framework is OPDA-native: an artefact of `ODR-0011 §8a` per S011/S012 framework convergence.

## Scheme header pattern

Per ODR-0011 §1a + §8a, every scheme carries 7 metadata fields:

| Field | Purpose |
|---|---|
| `skos:prefLabel` | Human-readable scheme label |
| `dct:title` | Long-form title (with disambiguation) |
| `skos:definition` | Scheme-level definition |
| `dct:source` | Authoritative source (regulator or ODR §) |
| `opda:ufoCategory` | UFO meta-vocabulary classification |
| `skos:scopeNote` | UFO + DOLCE citation |
| `opda:hasSteward` | Council steward (named) |

## Identity rule

Per ODR-0005 Anti-pattern §5: scheme members bind to corresponding OWL classes via `skos:exactMatch` — **NEVER** `owl:sameAs` (which would propagate identity-collapse across contexts under inference).

## Deprecation lifecycle

Per ODR-0011 §5a + ODR-0017, deprecated members carry `owl:deprecated true` + `dct:isReplacedBy` to the successor concept. The [`opda:DeprecationChainRule`](../shacl-af-rules.md#opdadeprecationchainrule) SHACL-AF rule materialises `opda:hasDeprecationStatus` ("with-succession" or "without-succession") into the validation report.
