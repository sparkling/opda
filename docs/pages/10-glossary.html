Standards & frameworks

TermFull nameRole
PDTFProperty Data Trust FrameworkOPDA's umbrella standard — JSON Schemas + OpenAPI + Trust Framework spec.
ToIPTrust Over IP FoundationLinux Foundation project. 4-layer governance model PDTF follows.
DIATFDigital Identity and Attributes Trust FrameworkUK government identity trust framework. DSIT-owned.
VCVerifiable CredentialsW3C data model for cryptographically verifiable claims. PDTF v2.0 is VC-based.
DIDDecentralized IdentifierW3C standard for self-sovereign identifiers. Used for PDTF participants.
VCDMVerifiable Credentials Data ModelW3C VC spec, versions 1.1 and 2.0 archived locally.
DIDCommDID CommunicationSecure-messaging protocol used between DID-identified parties.
OIDCOpenID ConnectIdentity layer over OAuth 2.0. Used in PDTF Auth pattern.
DCAMData Management Capability Assessment ModelEDM Council framework. Not the framework OPDA has chosen (ToIP is).
DAMA-DMBOKData Management Body of KnowledgeBroader data management reference. Component-level usable.
FAIRFindable, Accessible, Interoperable, ReusableData sharing principles.
SKOSSimple Knowledge Organization SystemW3C standard for taxonomies / controlled vocabularies.
OWLWeb Ontology LanguageW3C standard for ontologies.
SHACLShapes Constraint LanguageW3C standard for validating RDF graphs.
DCATData Catalog VocabularyW3C vocabulary for describing datasets.
RDFResource Description FrameworkW3C foundational data model for the semantic web.
JSON-LDJSON for Linked DataRDF serialisation in JSON. PDTF's bridge from JSON Schema to RDF.

Concepts (PDTF-internal)

TermDefinition
Trust FrameworkAgreements + rules + technical standards by which organisations exchange verified data.
Verified ClaimA data assertion with attached provenance — who said it, when, with what evidence.
OverlayAn optional extension to the base PDTF transaction schema (e.g. baspi5.json).
Property PackThe bundle of standardised property data that accompanies a transaction. HBSG initiative.
AccreditationOPDA's scheme for certifying members as trusted PDTF participants.
Trust RegistryAuthoritative list of who is authorised to do what within a trust framework.
