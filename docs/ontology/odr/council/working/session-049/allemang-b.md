## Q1 — Adopt a single canonical UFO typing axis corpus-wide, or keep gated/partial?

### Verdict: **REVISE** · Ballot: **AGAINST** (the corpus-wide form); **FOR** a gate-first, scoped form only.

**Rationale.**

**1. The proposition inverts the order of operations that session-042 settled.** The anti-rigid-subsumption CI gate is named as the *payoff* of corpus-wide typing — "unlock a gate." But session-042's whole ruling is that **the gate is the precondition, not the prize.** You ship the running check first; *that* event creates the consumer and converts the per-type type-assertion from decoration into the gate's input data. Re-typing 41 classes *in order to maybe author a gate later* is materialising the worksheet before anyone files a claim. The minimum-model rule (*Semantic Web for the Working Ontologist*, 3rd ed., Ch. 12 — model only what a consumer reads) forbids it in that order; it would *require* it in the reverse order. So my position is not "no" — it is **"gate first, and the gate dictates the scope."**

**2. "Single canonical axis" buys decidable rigidity for the lattice the gate ranges over — which is small.** ODR-0027/S036 gave OPDA a deliberately **flat** classification topology: kinds carry facets via `isMemberOf`/SKOS, subtype only for shared identity criterion. There is no deep subsumption tree. An anti-rigid-subsumption gate (`−R ⊐ +R` is the violation: an anti-rigid class subsuming a rigid one) only has work to do **where one class actually `rdfs:subClassOf` another.** On a flat lattice, that is a handful of edges, not 41 nodes. Typing the other ~33 classes that participate in *no* subsumption edge produces a vector no rigidity check ever reads — that is the session-042 "35 classes of unused vector" finding restated. Working-ontologist practice (*SWWO* Ch. 13, modelling-for-reuse): type the class where a reader will consume the type; don't type for symmetry.

**3. The honesty cost is real and the ADR concedes it.** ADR-0050 §Decision Drivers lists "Honesty — keep any upper-ontology anchor inert." Good. But corpus-wide UFO-sortal typing valued by a closed scheme is, today, **inert decoration on ~33 of 41 classes** — no entailment (OPDA's frozen 7-rule closure ignores `rdf:type gufo:X` for anything outside its safe group), no SHACL consumer, no gate. The handover's own prior-art citation is the rule here: ODR-0101, *"an unconsumed axiom is a latent liability, not a free asset."* A canonical typing triple on a class no gate ranges over is exactly such an unconsumed axiom. It does not lie — but it is maintenance debt with no reader, and that is what working-ontologist minimalism exists to prevent.

**4. What I will affirm.** A single canonical *axis definition* — one annotation property, one closed SKOS sortal scheme, a stated rule that any second axis is a derived view — is **sound design** and I'd adopt the *convention* tomorrow. My objection is to **corpus-wide population**. Define the axis; ship the anti-rigid-subsumption gate; populate the axis on **the subsumption lattice the gate ranges over plus its immediate contrast set** — identical scope to session-042's adjudicated outcome. That gives you a decidable, gated rigidity check *and* keeps the corpus free of unread type-assertions. That is the simplest model that actually does the work.

## Q2 — If adopted: mechanism + scope

### Verdict: **REVISE** · Ballot: **FOR** (one annotation property + closed SKOS sortal scheme); **AGAINST** corpus-wide scope, **FOR** gate-scoped phased.

**Mechanism — AFFIRM, with reuse over mint.**
One `owl:AnnotationProperty` (e.g. `opda:ufoSortalType`) valued by a **closed SKOS scheme** with `sh:in` governance is the right envelope — and OPDA **already has most of it**: ADR-0045 shipped `opda:UFOCategoryScheme` (nine concepts, four already carrying their OntoClean signature in `skos:definition`), and ADR-0046 shipped the OntoClean meta-property markup envelope. The reuse-vs-mint test (*SWWO* Ch. 10 — reuse existing vocabulary before minting) says: **do not mint a parallel sortal scheme.** Extend/rename the existing `UFOCategoryScheme` to be the canonical sortal axis and hang the new annotation property on it. Minting a second scheme beside it would create exactly the "two registers" redundancy Tom Baker flagged in 042 (DCMI one-to-one principle).

**Scope — REVISE to gate-scoped, phased.**
Not corpus-wide in one pass. The honest sequence:
1. **Define** the canonical axis (annotation property + closed scheme + "second axis is derived" rule) — cheap, no per-class cost.
2. **Ship** the anti-rigid-subsumption meta-shape as a running CI gate (TBox-only, the ADR-0046 / ODR-0031 R3 tag-guard pattern — never touches instance data).
3. **Populate** the axis on the lattice the gate ranges over + its contrast set, atomically with the gate (042's "(a) and (b) together or neither").
4. **Expand** only if and when a second named consumer needs a wider population.

This is ADR-0050's **option (C) Hybrid** — agent/relationship layer first (where the Role/RoleMixin/Relator subsumption edges actually live and where rigidity bites), expand corpus-wide only on consumer demand — *not* its option (A) corpus-wide-in-one-pass. Phased, gate-driven, reuse-first.

## Pre-emptive challenge to my Council B peers (Guarino especially)

Nicola — in session-042 you argued the full ±R/±I/±D/±U vector on all ~40 classes, with the sharp point that *"a partial vector yields an unsound completeness-check / false green — the worst CI failure mode."* I expect the same here: corpus-wide canonical typing so the anti-rigid gate is *sound*, not partial. My answer in 042 was the scope adjudication, and I'll put it to you again: a `−R ⊐ +R` check is **only unsound if it skips a class that participates in a subsumption edge.** Classes outside every subsumption edge cannot create an anti-rigid-subsumption violation, so omitting their type cannot produce a false green. Therefore "the lattice the gate ranges over + its contrast set" is **complete for the check** without being corpus-wide. Show me a `−R ⊐ +R` violation that this scope would miss but corpus-wide typing would catch, and I move toward you. Absent that, corpus-wide is over-population, not soundness.

## Final position (provisional, pre-rebuttal)

- **Q1:** REVISE / AGAINST corpus-wide, FOR gate-first-and-scoped. The gate is the precondition; the scope follows the gate's lattice.
- **Q2:** REVISE / FOR the one-annotation-property + closed-SKOS mechanism (reusing `UFOCategoryScheme`, not minting), AGAINST corpus-wide scope, FOR gated phased (= ADR-0050 option C).

Grounded throughout in *Semantic Web for the Working Ontologist* (3rd ed., Ch. 10 reuse-vs-mint, Ch. 12 minimum model, Ch. 13 modelling-for-reuse) and OPDA's own session-042 precedent.

## Final

**Posture:** Both questions REVISE. The proposition's headline — *"type every class on one canonical axis TO UNLOCK an anti-rigid-subsumption CI gate"* — is, against the committed corpus, **already substantially satisfied**, and its one expensive demand (corpus-wide rigidity re-typing of ~41 classes) is **declined by all three Council B voices** on the session-042 minimum-model precedent. What survives is a smaller, durable, gated invariant. Verified empirical anchors (checked at file:line, per the handover's "never trust green" warning):

- The anti-rigid-subsumption gate already ships green: check 8 `check_ontoclean_tbox` + check 9 `check_ontoclean_identity_tbox` (`three_graph_test.py:348`, `:417`), TBox-only, ADR-0046.
- The meta-shape is `sh:targetSubjectsOf opda:ontoCleanRigidity` (`shapes.py:507`) — so an untyped class is simply outside the target set; a partial axis does **not** make the gate unauthorable (refutes ADR-0050's pivotal premise on the mechanism).
- `opda:ufoCategory` (single canonical category axis, `owl:AnnotationProperty`, closed `UFOCategoryScheme`) is on **39/40** classes already; the lone residual subject is `opda:SpecialCategoryScheme`, a `skos:ConceptScheme` correctly **exempt by level** (a meta-level organising artefact, not a first-order domain universal — Guarino 1994, "The Ontological Level"), not a gap to close. The axis includes `Event` (5) and `Quality Value` (2), which are **not** endurants — so the invariant's scope predicate is "first-order domain universal," **not** "endurant universal" (the latter would misfire by exempting the perdurant/quality-value rows).
- `opda:ontoCleanRigidity`/`Identity` are emitted from a **hand-maintained 8-row dict** (`ufo_categories.py:300-308`) keyed by class local-name — **not** an automated function of `ufoCategory`, despite the `rdfs:comment` claiming derivation. On exactly **8 domain classes** (Buyer, Proprietor, Proprietorship, Relator, Role, RoleMixin, Seller, Transaction).
- Every intra-`opda:` `rdfs:subClassOf` edge is one of the 5 already in the gate; all other supers are external (`prov:`/`org:`/`vcard:`/`time:`/`skos:`). The lattice is **flat by ODR-0027 doctrine** (Davis's enumeration, accepted).

### Q1 — Adopt a single canonical UFO typing axis corpus-wide, or keep gated/partial?

**VERDICT: REVISE · BALLOT: AGAINST** (the proposition's corpus-wide *re-typing*); **FOR** the single-canonical-axis *invariant* as a stated, gated rule.

- I **concede Guarino's formal core**: rigidity must denote a *function* `Type → {+R,−R,~R,non-R}`; two unconstrained parallel typing axes make it a *relation*, and the −R⊐+R checker is then ill-typed — a failure *inside OPDA's own SHACL gate*, no reasoner involved (checker-totality, distinct from logical decidability). The single-canonical-axis principle is therefore a genuine, grounded precondition, not a stylistic preference. **FOR the invariant.**
- I **conceded, and Davis then bounded, Guarino's growth-frontier counterexample** (`CreditRiskAssessment ⊐ RiskAssessment ⊐ AssessorRole`): the gate goes silent on a future subsumption edge through an *untyped middle class*. This defeats a "type only today's lattice, fixed" reading. **But** the remedy it forces is **not** corpus-wide pre-population — it is (a) automating the category→rigidity projection so any sortal-categorised class auto-enters the gate's target set, and/or (b) an **edge-targeted meta-shape** that fails CI when an intra-`opda:` subClassOf edge has an endpoint lacking a rigidity projection. Both catch the violation *at edge-authoring time*; neither requires typing the ~31 classes that sit in no subsumption edge. ODR-0027's flat doctrine makes such edges rare and reviewable, so an edge guard is sufficient and corpus-wide pre-typing is over-insurance.
- **AGAINST corpus-wide rigidity re-typing**: ~31 of 41 classes participate in no intra-`opda:` subsumption edge; a rigidity tag on them exercises no checker on any commit, present or future — the unconsumed axiom session-042 forbade and ODR-0101 names a "latent liability, not a free asset" (cited in the handover itself). For the non-sortal categories (Information Object — the largest group, by the corpus's own definition an IAO documentary class — Quality, Quality Value, Collective), a coerced ±R bit would *misstate* their nature (Guarino's own caveat b).

**Net:** the corpus-wide *category* axis is already met; the *rigidity projection* stays scoped to the sortal subsumption lattice and is extended **by derivation/edge-guard, never by blanket assertion**. That is REVISE — more than "keep partial, do nothing" (it adopts and *enforces* the invariant), and far less than "re-type 41."

### Q2 — Mechanism + scope

**VERDICT: REVISE · BALLOT: FOR** (the one-annotation-property + closed-SKOS mechanism — *which already exists*; **reuse, do not mint**); **AGAINST** corpus-wide population scope; **FOR** gated/derivation-driven scope.

**Mechanism — FOR, by reuse not mint.** One `owl:AnnotationProperty` (`opda:ufoCategory`) valued by the closed nine-term `opda:UFOCategoryScheme`, inert (OWL 2 §10.1), annotation-graph-quarantined, with `ontoCleanRigidity`/`Identity`/`Dependence` as **derived projections** and the TBox meta-shape as consumer — this is ADR-0045 + ADR-0046 **as shipped**. The canonical axis must be the `ufoCategory` SKOS scheme; the `gufo:` `rdf:type` markers (ADR-0034) remain an aligned/derived view via the existing `skos:closeMatch`, **not** a competing second rigidity source. Reuse-vs-mint (*SWWO* 3rd ed., Ch. 10): do **not** stand up a parallel sortal scheme or revive an `opda:Role`/`opda:Relator` meta-*class* tree as an independent typing apparatus — that is the two-axis problem in miniature and the DCMI one-to-one redundancy Baker flagged in 042.

**Scope — REVISE to derivation-driven + edge-guarded, never a blanket re-type.** Adopt, atomically (ships-green-or-not-at-all — the session-042 atomicity bar I co-sign with Davis):
1. **State the invariant** — a §rule in the doctrine/ODR: `ufoCategory` is the single canonical typing axis; `ontoCleanRigidity`/`Identity`/`Dependence` are `g(ufoCategory)` projections, never independent assertions; any second axis is a derived view.
2. **Axis-consistency meta-shape** over the **8 already-tagged classes** — fails CI if a class's `ontoCleanRigidity`/`Identity` contradicts its `ufoCategory` signature. Consumer = the maintainer who would otherwise let the two axes drift (Kendall's session-042 "named consumer #1 — internal, certain, fires on every edit"; the warrant that flipped Baker). The "derived-from" rule is already documented intent (`opda-annotations.ttl:124,130`) — what is missing is the *gate that enforces it*.
3. **Growth-frontier guard — REQUIRED:** an edge-targeted meta-shape failing CI when an intra-`opda:` `rdfs:subClassOf` edge has an endpoint lacking a rigidity projection. This is the *minimal sufficient* fix for soundness — it catches Guarino's counterexample at edge-authoring time **whether or not** the projection is automated. (Davis's DA scoping, accepted: the 8 hand-values are correct in *extension* but the docstring lies in *mechanism*; the defect is **latent** — it bites only when a sortal subkind edge is added and a maintainer trusts the docstring. (b) guards that directly.)
4. **Automate the projection — RECOMMENDED (not a soundness precondition):** replace the hand-maintained 8-row dict with a real emitted `g(ufoCategory)` (the 9 `UFOCategoryScheme` signatures are the function table). This makes single-valued-ness hold *by construction* and stops the `:124/:130` docstring asserting a derivation the code does not perform — but its consumer ("a maintainer who reads the docstring") is weaker than (3)'s ("prevents a silent unsound edge"), so it is honest hygiene, not mandatory. **Materialise, do not derive on-the-fly** (on-the-fly is gate-complete but signal-lossy — the rigidity values never become queryable/AI-legible triples).

**Net mandatory set (atomic-or-nothing):** (1) invariant STATED + (2) axis-consistency meta-shape over the 8 + (3) edge-targeted guard. **Recommended:** (4) automate `g(ufoCategory)`. Tag-count delta: **zero** either way; the ~31 stay untyped.

**Decline:** corpus-wide rigidity-vector population, any minted parallel scheme, and any ±R/±I coercion onto non-sortal categories.
