## OPENING

### The proposition's headline deliverable already shipped. Three months ago. Read your own corpus.

The proposition is: *"type EVERY OPDA class on ONE canonical UFO ontological-type axis … to make OntoClean rigidity decidable and **unlock an anti-rigid-subsumption CI gate**."*

That gate is not hypothetical. It is **running, green, in the committed tree right now**, built by ADR-0046 and ratified by the operator on 2026-06-16:

- `OntoCleanAntiRigidSubclassing_MetaShape` — the canonical *"a rigid type MUST NOT be `rdfs:subClassOf` an anti-rigid one"* check — emitted in `opda-shapes.ttl` (`emitters/shapes.py:507`), consumed by **check 8** `check_ontoclean_tbox` (`ci/three_graph_test.py:348`).
- `OntoCleanIncompatibleIdentitySubclassing_MetaShape` — the ±I sibling, *"a `supplies-IC` type MUST NOT subsume another `supplies-IC` type"* — **check 9** `check_ontoclean_identity_tbox` (`ci/three_graph_test.py:417`), added 2026-06-17 (yesterday).
- Both run over the class+annotation graph, **TBox-only, never instance-keyed** (ODR-0031 R3 tag-guard), both empty against the corpus, both with non-vacuity proven by synthetic positive-control unit tests (`test_ontoclean_identity_supplies_subclasses_supplies_fails` + three negative controls). All nine three-graph gates green; byte-identity re-pinned.

So the proposition's *entire stated payoff* — "unlock an anti-rigid-subsumption CI gate" — **is already unlocked, and it did not require typing every class.** It required typing exactly the eight classes the check ranges over: `Relator`, `Transaction`, `Proprietorship`, `Role`, `RoleMixin`, `Proprietor`, `Buyer`, `Seller` (verified, `opda-annotations.ttl:154–259`). That is the lattice the gate traverses *plus its contrast set*, and not one class more.

This is dispositive for Q1. **You cannot adopt a corpus-wide re-typing to unlock a gate that is already green over the partial typing.** The benefit column of ADR-0050's own consequences ("a single canonical axis would … unlock an anti-rigid-subsumption CI gate") is **factually spent**. What remains on the table is not a benefit; it is ~31 more classes of tagging that no check reads.

### The corpus already HAS a single axis. It's `opda:ufoCategory`, and it's on 39/40 classes.

The proposition is framed as if OPDA is in some primitive "mostly untyped" state needing a first canonical axis. It is not. `opda:ufoCategory` — an `owl:AnnotationProperty`, string-valued, annotation-graph-quarantined, behind the sixth CI gate (ADR-0045, session-041) — **already carries a single ontological-category value on essentially every class**: `Substance Kind`, `Relator`, `Role`, `RoleMixin`, `Information Object`, `Quality Value`, … (verified across `opda-annotations.ttl`). That *is* "one canonical axis." It already exists, it is already inert, it is already governed by `sh:in`, it already dereferences at `/pdtf/ufoCategory`, and it already has read-side projection pages.

So what is the proposition actually asking for, net of what ships? Strip the framing and it is one of two things, both small and both already adjudicated:

1. **Promote the string value to a closed UFO-sortal SKOS scheme** — *exactly* the "mint a local SKOS scheme" move Allemang voted AGAINST in session-041 ("the textbook over-modelling error my own book names"), that the council adopted only in the bounded **nine-`closeMatch`-edges** form, never as a per-class re-typing. Q2's "one annotation property valued by a closed UFO-sortal SKOS scheme" is **already the shipped design** for the nine categories; extending the *value set* to be SKOS does not require re-typing 40 classes.
2. **Extend the ±R/±I OntoClean vectors from 8 classes to all ~40** — *exactly* Guarino's "full-vector, all ~40" position that session-042 **rejected** in favour of Allemang's "the lattice the gate ranges over + its contrast set." The council has already ruled, 4–0, that the blanket vector "over-models (~30 unread vectors)" (ADR-0046 Option B, rejected).

Every sub-move the proposition decomposes into has been put to this council and **declined in its corpus-wide form within the last three months.** Council B is, with respect, re-litigating sessions 040/041/042 under a new headline.

### The "decidable rigidity needs a single-valued axis" premise is technically misapplied here.

ADR-0050's pivotal claim is: *"OntoClean rigidity must be single-valued per class or the meta-property is undecidable and an anti-rigid-subsumption CI gate cannot be authored against an ambiguous axis."*

Two problems.

**First — OPDA does not reason. The word "decidable" is borrowed from a regime OPDA opted out of.** Per the standing doctrine (ODR-0025/0026/0029; the 2026-06-17 handover's gap-#15 doctrine; my fellow-traveller memory `[[opda-entailment-domain-range-as-signal]]`), every UFO/OntoClean tag is **documentary AI-signal, NEVER entailed**; the only inference is the frozen 7-rule RDFS-Plus closure, which the tags are quarantined out of. "Decidability" is a property of a *reasoner's* satisfaction problem. OPDA runs no reasoner over these tags. The thing the proposition calls "undecidable rigidity" is not a logical defect — it is simply *classes that haven't been hand-annotated yet*, and the gate that would catch a contradiction (check 8/9) **only ranges over annotated classes and already passes.** There is no undecidability to cure because there is no decision procedure running.

**Second — the gate that exists is sound on a partial axis precisely because the meta-shape is `sh:targetSubjectsOf` the tag.** It fires on a class *iff* that class carries `opda:ontoCleanRigidity`. An untyped class is simply outside the gate's target set; it does not "ambiguate" anything. ADR-0046 went out of its way to prove this with positive/negative controls. The premise that a partial axis makes the gate unauthorable is **refuted by the running gate.** The author of ADR-0050 appears not to have checked whether the gate they want already exists. It does.

### Cost is real, recurring, and lands on the modeller — for an audit of a flat lattice.

ADR-0050's own driver names the cost: *"adopting touches the whole class hierarchy and the gUFO/Relator/Role apparatus."* Concretely, corpus-wide adoption means:

- Hand-assigning a defensible ±R/±I (and the canonical UFO sortal) to ~31 more classes — `RegisteredTitle`, `RiskAssessment`, `RoomDimension`, `Search`, `Survey`, and the rest of the descriptive/information-object family. These are the classes session-029 and session-042 *deliberately left untyped* because their category "was never in question" (ADR-0046, Allemang) or because they straddle (the `priceQualifier`/`ownershipType` family, session-029).
- A **standing maintenance tax**: every new class, every hierarchy edit, now must carry a full vector or fail a corpus-wide single-axis gate — Guarino's own "a *partial* vector yields an unsound completeness-check / false green" cuts *against* corpus-wide adoption, because the moment you mandate the axis corpus-wide you own the obligation to keep all 40+ cells correct forever, on pain of red CI, for a lattice that ODR-0027 keeps deliberately **flat** (classification-over-inheritance — there is almost no subsumption depth for the rigidity check to even traverse).

W3C DWBP BP 1 is the discipline here: *provide metadata that meets a need.* The need (the anti-rigid gate) is met by 8 tags. The other 31 meet no consuming need. Under my BBC `/programmes/` experience this is exactly the trap: a backbone built for completeness, dereferenced by nothing, that becomes a liability the moment the model grows. The handover's own prior-art citation says it: *"an unconsumed axiom is a latent liability, not a free asset"* (ODR-0101, quoted at handover line 87). Corpus-wide re-typing manufactures ~31 unconsumed axioms.

### What I am FOR (the honest alternative).

**(B) Keep the gated/partial typing.** It already delivers the proposition's stated benefit. If the operator wants the *cosmetic* completeness of a value on every class, that is **already shipped** as `opda:ufoCategory` on 39/40 — and the one honest, cheap improvement available is to close the 1-class gap and to promote the string value to the already-designed SKOS scheme **without** mandating a corpus-wide ±R/±I vector or a corpus-wide single-axis gate. That is a value-set refinement, not a re-typing programme, and it does not cross the YAGNI line.

I will hear Guarino's soundness case and Allemang's framing, but my opening position is: **the gate is green, the axis exists, the cost is for nobody, and three prior councils already drew this exact line.**
