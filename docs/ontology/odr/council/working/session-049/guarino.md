3. **The anti-rigid-subsumption gate the proposition wants to "unlock" ALREADY RUNS.** `three_graph_test.py` **Check 8** (ADR-0046, `status: accepted`) runs the canonical OntoClean check over the class+annotation graph: the violation form `?sub ontoCleanRigidity "rigid" . ?sub rdfs:subClassOf ?super . ?super ontoCleanRigidity "anti-rigid"` **must return empty**. **Check 9** runs the ±I sibling (two own-IC suppliers in a subsumption). ### Q1 — Adopt a single canonical UFO typing axis corpus-wide, or keep gated/partial?

**VERDICT: REVISE. BALLOT: FOR** (FOR the single-axis *principle*, corpus-wide, as the canonical invariant; REVISE because the corpus already realises it for the spine and the extension must be *derivation-complete, not assertion-blanket*).

**The formal case — why single-valued ontological typing is the precondition for decidable rigidity.**

Rigidity is a meta-property of a *property* (a universal/type) Φ, defined modally (Guarino & Welty, "Evaluating Ontological Decisions with OntoClean," *CACM* 45(2), 2002; "An Overview of OntoClean," *Handbook on Ontologies* 2nd ed., 2009, §3):

- Φ is **rigid (+R)** iff necessarily every instance of Φ is necessarily an instance of Φ (∀x □(Φx → □Φx)). *Person, Dog.*
- Φ is **anti-rigid (−R)** iff necessarily every instance of Φ is *possibly not* an instance of Φ (∀x □(Φx → ◇¬Φx)). *Student, Buyer, Seller — every student can cease being one.*
- Φ is **semi-rigid (~R)** iff rigid for some instances, anti-rigid for others.
- Φ is **non-rigid** iff not anti-rigid and not rigid for all.

The OntoClean subsumption constraint that the gate enforces is a **theorem**, not a convention (Guarino & Welty 2009, §3, the rigidity-incompatibility constraint): **an anti-rigid property cannot subsume a rigid one** (−R ⋣ +R). Proof sketch: if +R-type R ⊑ −R-type A, then every R-instance is an A-instance; A anti-rigid means each A-instance possibly-not-A; but R rigid means each R-instance necessarily-R hence necessarily-A — contradiction with possibly-not-A. So `Person ⊑ Student` is incoherent; `Student ⊑ Person` is sound.

**Now the decidability claim, stated precisely.** A *checker* for this constraint is a function over the subsumption edges that, for each edge ⟨sub, super⟩, reads the rigidity value of `sub` and of `super` and tests the forbidden combination. **That function is well-defined iff rigidity is single-valued per type.** If a class carries *two unconstrained parallel typing axes* — say `ufoCategory = Kind` (→ +R) on one axis and an independent `role-status = role` (→ −R) on a second axis with no derivation rule binding the two — then `rigidity(C)` has **no functional value**: the checker reads `+R` along axis-1 and `−R` along axis-2 for the *same* C, and the forbidden-combination test is **ill-typed** — it cannot return a verdict, because its input is a *relation*, not a *function*. This is not a performance problem or an ambiguity to be resolved by precedence; it is that the predicate `rigidity : Type → {+R, −R, ~R, non-R}` **fails to denote a function** the moment two axes assign incompatible values to one type without a governing derivation. **That is the formal sense in which two unconstrained parallel axes make rigidity undecidable: the meta-property ceases to be a function, and no total checker exists over a non-functional input.**

The cure is precisely the proposition's invariant: **one canonical axis; any second axis is a *derived view* of the first** (a function *of* `ufoCategory`, not an independent assertion). The instant the second axis is `f(ufoCategory)` rather than free, rigidity is single-valued again — because it is `rigidity = g(ufoCategory)`, a composition of functions, total and decidable. **This is exactly what the corpus does today** (`ontoCleanRigidity` "Derived from each type's `opda:ufoCategory` signature" — `opda-annotations.ttl:133`): the rigidity axis is not parallel to the category axis, it is *projected from* it. That is why Check 8 *can* run: its input is a function.

**What breaks under two unconstrained parallel typing axes** — concretely, the three failure modes:
1. **Non-functional rigidity → no checker.** As above: the −R⋣+R gate is undefined on its inputs. This is the headline.
2. **Silent false-greens / false-reds.** If a precedence rule is bolted on *ad hoc* (axis-1 wins), the corpus can ship a class that is `Kind` on axis-1 (the gate sees +R, passes) while its *real* modelling role on axis-2 is anti-rigid — the gate is green but the lattice is unsound. A partial or contradictory vector is, as I have always held, **worse than no vector: it yields an unsound completeness-check** (this was my session-042 Q2 position and it holds here a fortiori).
3. **Loss of the identity-inheritance discipline.** Rigidity is not the only meta-property keyed off the canonical axis — **identity (±I)** is too (Check 9). OntoClean's deepest constraint is that *a sortal supplies exactly one identity criterion and a subtype inherits it* (Guarino & Welty 2009, §3; the IC-carrying/IC-supplying distinction). Two parallel axes that disagree on whether C `supplies-IC` or `carries-IC` make the IC-incompatibility checker non-functional in the same way. Identity is the *more* fundamental meta-property (rigidity without identity is under-determined — DOLCE, Masolo et al., WonderWeb D18, 2003, §2), so the single-axis precondition is doubly forced.

**Therefore FOR the single-axis invariant corpus-wide.** It is the formal precondition for *any* OntoClean meta-property gate (rigidity *and* identity), not a stylistic preference. A class's ontological nature — its category in the rigid-sortal / anti-rigid-role / relator / quality lattice — is a *single fact*, and representing it on one canonical axis is simply representing that fact once. Multiple unconstrained axes assert that a type has two incompatible natures, which is not pluralism — it is a category error in the literal OntoClean sense.

**Why REVISE and not plain AFFIRM.** Three formal qualifications the proposition's "type *every* class" must absorb, or it over-claims:

- **(a) The extension must be derivation-complete, not assertion-blanket.** Corpus-wide single typing means *every class carries a canonical `ufoCategory`* (≈39 do; the remainder must) **and** *every meta-property is `g(ufoCategory)`* — never a second hand-assertion. The ADR-0046 rigidity tags are derived for the lattice; extending corpus-wide means extending the *derivation*, not hand-writing ±R on 30 more classes (that re-introduces failure-mode 2). The single axis is `ufoCategory`; rigidity/identity/dependence are *projections*, emitted only where the projection is not already inherited `category → scheme → signature` (my session-042 redundancy rule).

- **(b) Three categories are not endurant universals and must be typed honestly, or the rigidity function is given garbage inputs.** I flag, against my own enthusiasm for completeness:
  - **`Information Object`** (12 classes — the *largest* group) is, by the corpus's own `skos:definition` (`opda-annotations.ttl:60`), *"an OPDA domain category rather than a core UFO endurant universal — cf. IAO information content entity."* Its rigidity is *not* a UFO-endurant fact; it is an IAO/documentary classification. Forcing it onto the *same* rigidity projection as `Kind`/`Role` would assert a modal-rigidity judgement OntoClean never licensed for information artefacts. Type it on the axis (it has a `ufoCategory`), but its rigidity cell may legitimately be `non-rigid`/undetermined, **not** coerced.
  - **`Quality` / `Quality Value`** descend from DOLCE's Quality/Quale/Region (`:68`, `:77`; D18, 2003), not from the sortal lattice. Qualities are rigid *as qualities* but they are not *sortals* and do not enter the −R⋣+R sortal-subsumption check the way Kinds and Roles do. The check ranges over **sortals**; quality individuals are a different square of the OntoClean grid.
  - **`Collective`** (1 class) is rigid but its identity criterion is *membership-based* (extensional/intensional collectives, Masolo et al. 2004) — a distinct IC family. Fine to type; do not assume its ±I cell matches a Kind's.

  The formal point: **the single axis must classify into the *full* OntoClean grid (sortal/non-sortal × rigid/anti-rigid/semi-rigid; +I/−I; +D/−D), not flatten everything to a rigidity bit.** A corpus-wide rigidity *value* is sound only where the category is a sortal universal; for the non-sortal categories the canonical axis still applies (you record *what they are*) but the rigidity projection is category-appropriate, possibly "non-rigid" or "n/a", never a coerced +R/−R.

- **(c) Inert is non-negotiable and already structurally guaranteed — keep it.** The whole apparatus is `owl:AnnotationProperty` (OWL 2 §10.1), annotation-graph-only, behind the three-graph quarantine; the meta-shapes are `sh:targetSubjectsOf`/SPARQL over the TBox, **never** instance-keyed (`opda-shapes.ttl:67`). This is correct and I will not trade it. Rigidity is a *modal* meta-property; OPDA cannot evaluate modal logic in OWL 2 DL (Almeida, Guizzardi, Sales & Fonseca, *gUFO*, 2026 — UFO's modal apparatus is not OWL-2-DL-expressible), so the axis *must* be documentary signal validated by a TBox meta-shape, exactly as built. The single-axis adoption changes the *scope* of the derivation, not the inertness regime.

### Q2 — If adopted: mechanism + scope?

**VERDICT: REVISE. BALLOT: FOR** (FOR one annotation property valued by a closed UFO-sortal SKOS scheme, inert — *which already exists*; REVISE the scope to **phased, derivation-driven**, not a flag-day blanket re-type).

**Mechanism — FOR, and it is already the corpus mechanism.** One annotation property (`opda:ufoCategory`) valued by a closed UFO-sortal SKOS scheme (`opda:UFOCategoryScheme`, nine concepts, each carrying `skos:closeMatch` to its gUFO type and its OntoClean signature in `skos:definition`), inert (`owl:AnnotationProperty`), annotation-graph-only, with the derived meta-properties (`ontoCleanRigidity`/`Identity`/`Dependence`) as projections and the TBox meta-shape as the consumer. **This is precisely ADR-0045 + ADR-0046 as shipped.** The "mechanism" question is therefore *already answered FOR* — my vote ratifies the existing design as the canonical single axis and rejects any *second, parallel* typing apparatus (e.g. a separate `opda:Role`/`opda:Relator` meta-*class* tree used as an independent rigidity source — see the well-foundedness note below). One subtlety I require: the canonical axis must be the **`ufoCategory` SKOS scheme**, and the `gufo:` `rdf:type` markers (ADR-0034, 5 descriptive leaves, `opda-descriptive-annotations.ttl:31–47`) must remain **a derived/aligned view** (`skos:closeMatch` already does this), *not* a competing second axis. If both `gufo:Quality rdf:type` and `ufoCategory "Quality"` are read as independent rigidity sources, that is the two-axis problem in miniature — bind them (the `closeMatch` edge is the binding; keep it the single source of truth, with `ufoCategory` canonical and `gufo:` the alignment projection).

**Scope — REVISE to phased + derivation-driven (this is the real decision).**

I withdraw nothing from my session-042 soundness principle (*a partial vector false-greens a completeness check*), but I apply it correctly to *this* question. The corpus-wide adoption is sound **iff** it is done as **completing a single derivation**, in phases that each leave the rigidity/identity *functions* total over their domain:

- **Phase 1 (DONE — ratify):** the Relator/Role spine + contrast set (8 classes), ±R/±I floor, ±D where a Relator decision turned on it. Check 8/9 green. This is ADR-0046.
- **Phase 2 (the proposition's substance — adopt):** complete the **canonical `ufoCategory` axis to 100% of classes** (the ~remaining classes that lack a tag), so that *every* class has its single ontological-type fact recorded. This is cheap, sound, and pure single-axis hygiene — no rigidity coercion required, because the *category* is the canonical datum and the rigidity projection follows only where the category is a sortal.
- **Phase 3 (guarded):** extend the **derived rigidity/identity projection** beyond the spine *only* into the sortal categories (Substance Kind subkinds, the genuine Roles/RoleMixins outside the spine) where the −R⋣+R and IC-incompatibility checks gain real subsumption edges to range over. For the non-sortal categories (Information Object, Quality, Quality Value, Collective) record the category but emit a rigidity cell only where it is a genuine modal fact, marked category-appropriately. **Never blanket-assert ±R on a class whose category does not determine it** (failure-mode 2).

**The decisive scope test (formal):** extend the rigidity projection to a class C **iff** C participates in a `rdfs:subClassOf` edge that the −R⋣+R or IC-incompatibility check ranges over (i.e. C has a sortal sub/superclass whose rigidity could collide). A rigidity tag on a class with no subsumption neighbour exercises no checker — it is the "unread decoration" Allemang and Baker rightly forbade in session-042, and I agreed. So: **single canonical *category* axis = corpus-wide (Phase 2); derived rigidity/identity projection = the subsumption lattice it ranges over, grown phase by phase (Phase 3).** This is *not* the gated/partial status quo (option B): it commits to the single-axis invariant for *every* class and to a *complete derivation*; it simply refuses to fabricate rigidity values where no checker reads them and where the category is non-sortal.

**Anti-rigid-subsumption CI gate (the "unlock"):** already shipped (Check 8). Corpus-wide adoption *widens its reach* (more sortal subsumption edges fall under it as Phase 3 proceeds) and adds the IC check's reach (Check 9). I record, formally, that the gate's *soundness* depends entirely on single-valued rigidity — so the gate and the single-axis invariant are the same commitment, and voting FOR the gate's extension is voting FOR the invariant.

2. **The derived projection is NOT actually derived — it is a hand-table (allemang-b's find, independently verified).** `_ONTOCLEAN_TAGS` (`ufo_categories.py:298-315`) is a hand-maintained dict of **exactly 8 entries** keyed by class local-name; the docstrings (`:323`, `:334`) *claim* "Derived from each type's `opda:ufoCategory` signature" but **nothing computes it**. This is the documented-intent-vs-emitted-bytes gap that produced the session-047 error, unfixed in the generator. It **vindicates the formal core**: single-valued rigidity holds today only by a hand-table that asserts a derivation it does not perform; the gate's coverage is pinned to 8 names, **decoupled** from the (already-universal) category axis.

### Q1 — Adopt a single canonical UFO typing axis corpus-wide, or keep gated/partial?

**VERDICT: REVISE. BALLOT: FOR.**

FOR the **single canonical-axis invariant**, corpus-wide, as the standing rule — because single-valued ontological typing is the **formal precondition for decidable rigidity AND identity**: the meta-property `rigidity : Type → {+R, −R, ~R, non-R}` must denote a **function**, and two unconstrained parallel typing axes make it a *relation*, over which no total checker (the −R⋣+R gate, Check 8) exists — its test becomes ill-typed on contradictory inputs (Guarino & Welty, *CACM* 45(2), 2002; *Handbook on Ontologies* 2nd ed., 2009, §3). This is **checker totality**, a property of the SHACL gate OPDA *does* run — not logical decidability, which OPDA correctly opted out of (the modal apparatus is not OWL-2-DL-expressible: Almeida, Guizzardi, Sales & Fonseca, *gUFO*, 2026). The single-axis rule is what keeps the gate's own function total for **class N+1**, not a property of the 40 that exist.

REVISE — and the revision is the whole substance, because the proposition-as-framed ("re-type *every* class on one axis to *unlock* a gate") is built on two stale/false premises that all three Council B voices reject:
- **The gate is already unlocked.** Check 8 (`check_ontoclean_tbox`) + Check 9 (`check_ontoclean_identity_tbox`) run TBox-only, green, with positive/negative controls (ADR-0046, accepted). The "unlock" payoff is **spent**. **REJECT** that premise.
- **Corpus-wide rigidity-VECTOR population is over-modelling.** Blanket ±R/±I on ~31 more classes is the "full-vector all-40" that session-042 rejected, and it would *coerce* a modal rigidity bit onto non-sortal categories — Information Object (the 12-class plurality, "an OPDA domain category rather than a core UFO endurant universal" by the corpus's own `skos:definition`), Quality/Quality Value (DOLCE, not sortals — Masolo et al., WonderWeb D18, 2003), Collective (membership-IC). **REJECT** that, unanimously with davis and allemang-b.

**What the single-axis adoption ACTUALLY is** (the co-signed disposition):
1. **Ratify the invariant**, scoped to every class that denotes a **first-order domain universal** — i.e. an entity in the UFO grid: Substance Kind, Role, RoleMixin, Relator, Quality, Quality Value, Event/perdurant, Collective, Information Object. Every such class carries exactly one canonical `ufoCategory`; any second typing view (the `gufo:` `rdf:type`/`skos:closeMatch` alignment; the OntoClean projection) is a **derived view of it, never an independent axis**. **Meta-level / organising artefacts** (`skos:ConceptScheme` and the like — e.g. `opda:SpecialCategoryScheme`) are **exempt by LEVEL** (Guarino 1994 — a scheme is a container of concepts, the same level as `UFOCategoryScheme` itself), and carry none. [Already true, 39/39 eligible — and the scope predicate is *level*, NOT endurance: it must ADMIT the 5 Event (perdurant) + 2 Quality-Value rows, which do and should bear a category; allemang-b's wording fix, accepted — "endurant universal" would wrongly exempt them.]
2. **Automate the derived view.** Replace the 8-row hand-table with a genuine emitted **function g : ufoCategory → (rigidity, identity, dependence)**. The table already exists as data — the nine `UFOCategoryScheme` `skos:definition` signatures (RelatorCategory +R+I+D; Role/RoleMixinCategory −R+D; SubstanceKindCategory +R+O; …). Wiring it makes single-valued-ness hold **by construction** (rigidity = g(ufoCategory), a composition of functions) and makes the gate's coverage track the universal category axis automatically.
3. **Guard-rail on g (caveat b):** g returns the **full OntoClean grid**, not a rigidity bit. Non-sortal categories get rigidity "non-rigid"/n-a — never a coerced ±R. A rigidity *value* that arms the −R⋣+R check is emitted **only where the category is a sortal** (Substance Kind, Relator, Role, RoleMixin + subkinds). Corpus-wide *derivable*; sortal-scoped *armed*. This is the same scope session-042 reached — now reached by a function, not a hand-table. No unread vector; no silent gate on the growth frontier.
4. **Scope of the rigidity/identity projection (davis's held refinement, pinned with my contrast-set sharpening):** a vector is authored **IFF the class participates in an intra-`opda:` `rdfs:subClassOf` edge as either party** — which automatically pulls in *both* endpoints (the −R⋣+R and ±I checks each need the subsumed *and* the subsuming class typed, or the test is half-fed). Today: the 5 intra-OPDA edges = 8 classes (the ADR-0046 set, already both-ends-closed). Tomorrow: whatever new intra-OPDA sortal edge appears, scoped in *when it appears*. Never on a class in no intra-OPDA edge (the unconsumed-axiom case, ODR-0101). **Two distinct scopes:** the *category* axis (clause 1) is broad — every first-order domain universal; the *rigidity projection* (this clause) is narrow — sortal categories on intra-OPDA subsumption edges. They differ because they answer different questions (what a thing *is* vs. whether its rigidity can collide in a subsumption).
5. **Enforce, atomically** (davis's bar, which I hold): two TBox meta-shapes — (i) a **consistency** check: the projection matches the category signature (g) on every tagged class; (ii) an **edge-targeted growth-frontier guard**: fail CI if any intra-`opda:` `rdfs:subClassOf` edge has an endpoint whose sortal category lacks its (derived) rigidity value. Together they make single-valued-ness hold *and* the gate non-silent on the frontier, by construction — ships green in the same change-set or nothing ships.

**Grounded:** Guarino & Welty 2002/2009 (rigidity/identity meta-properties; the −R⋣+R and IC-incompatibility theorems; methodology evaluated over the taxonomy *as extended*); Guarino 1994 ("The Ontological Level" — the level boundary that exempts scheme artefacts); Masolo et al., WonderWeb D18, 2003 (DOLCE Quality/Quale/Region — the non-sortal quality categories); Almeida et al., *gUFO*, 2026 (why the axis must be inert documentary signal, not reasoned).
