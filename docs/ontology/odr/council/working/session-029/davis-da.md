# Session 029 (R2 reconvened) ‚Äî Ian Davis, Devil's Advocate

**Lens:** pragmatic linked-data minimalism ‚Äî RDF/Turtle vocabulary design, follow-your-nose dereferenceability, and the hard rule that one does not mint structure (classes, namespaces, modules) without a demonstrated consumer need.

## Opening ‚Äî the case against spawning

In Session 025 I held against minting anything until the ¬ßQ2a(a) trigger fired over a *counted, UFO-typed leaf-set* plus *a named load-bearing query per axis*. The gating event has now landed ‚Äî and it did not produce that artefact. The curated walk (ODR-0024) typed 164 leaves by **bearer Kind** (`rdfs:domain`), not by UFO meta-category: ODR-0024 R1 explicitly distributes leaves to `opda:Property` / `opda:LegalEstate` / `opda:Person` / `opda:Search` / `opda:Valuation` / `opda:Transaction` / `opda:RiskAssessment`, and R5/R12 say in terms that the emitted terms "are already correct G-grade in the TTL" with **no** Quality/Mode/Substance-Kind partition computed anywhere. The precise input ¬ßQ2a(a) and Session 025 anticipated ‚Äî a UFO-axis partition ‚Äî was never built. What we have is a bearer-domain partition wearing the walk's coat. To spawn three permanent w3id.org namespaces around an analytic meta-category the corpus has not computed, on the strength of zero tabled consumer queries, is exactly the premature commitment the subtractive programme forbids ‚Äî and exactly what I warned against. The gate-override that let Session 025 convene discharged the *convening* gate, not the *substantive* ¬ßQ2a(a) trigger (recorded, S025 ‚öÝ). That trigger is still unfired. I argue REJECT-the-UFO-split / DEFER, and I will state precisely what would turn me.

## Q1 ‚Äî Load-bearing? **REJECT**

No named consumer query has been tabled ‚Äî not in ODR-0023 R2, not in Session 025, not in ODR-0024. Allemang's own ratified criterion (S025) is the governing test and it is *mine* lifted to modules: "a sub-module spawns only when a named consumer query needs an entailment that holds for that axis and not the others ‚Äî turning the axis off must change an answer, else it is decorative and MUST NOT spawn." Turning the Quality/Mode axis off today changes **no** answer, because no query consumes a Quality-vs-Mode entailment ‚Äî consumers reach `currentEnergyRating` or `groundRent` by predicate and bearer, follow-your-nose, never by UFO meta-type. With zero leaves UFO-classified (ODR-0024 R12), "the a/b/c spawn would declare what we cannot derive" (Allemang, S025) ‚Äî load-bearing is unmet on the panel's own threshold.

## Q2 ‚Äî Valid partition key? **REJECT**

UFO meta-category is not a rigid, stable partition key fit for a permanent module boundary, and the binding table proves it on the corpus's own flagship cases. ODR-0008 ¬ßQ5a tags `priceQualifier` as "**Mode / Quality Value**" ‚Äî one leaf, two axes ‚Äî so a Quality module and a Mode module would *both* claim it or *neither* would; a w3id.org boundary cannot be drawn through a leaf that straddles. `ownershipType` re-sorts the other way: ¬ßQ5a and the ODR-0024 R1 register both put it as a Quale-in-Region whose `rdfs:domain` is `opda:LegalEstate` ‚Äî is it a `property-quality` (Allemang's axis) or a `legal-estate-attribute` (Kendall's)? The split's own stewards would contest the same leaf. A partition key whose members re-sort under examination is decorative, not rigid; minting namespaces around it bakes an unstable analytic call into permanent IRIs that consumers must then dereference forever.

## Q3 ‚Äî Module reality. **REJECT (no module is real)**

ODR-0008c collapses to a one-line query. The 25 LegalEstate leaves are *already* partitioned by `rdfs:domain opda:LegalEstate` in the emitted TTL (ODR-0024 R1: "`opda:LegalEstate` tenure/lease/charge attrs"); "leaves whose domain is LegalEstate" is `SELECT ?p WHERE { ?p rdfs:domain opda:LegalEstate }` ‚Äî re-badging an existing domain assertion as a module buys nothing dereferenceable. ODR-0008a/b are worse: they require a fresh per-leaf Quality-vs-Mode gUFO typing pass over the 69 `opda:Property` leaves that **the walk did not perform** (ODR-0024 typed by bearer, R12) ‚Äî so spawning a/b now mints the namespace *before* the content that would justify it exists, inverting need and structure. And the split addresses at most the ~94 Property+LegalEstate leaves; the other ~70 sit on classes it does not touch ‚Äî authority-retrieved artefacts already routed to ODR-0008d (R1/¬ßQ2a(b)), agents to ODR-0006, transactions to ODR-0007 (ODR-0007/ODR-0008d own these). A three-axis split that covers under 60% of the emitted leaves and leaves the rest to other modules is not a clean modularization; it is a partial overlay.

## Q4 ‚Äî Disposition. **REJECT the UFO-axis split; DEFER with a sharpened trigger**

Keep ODR-0008 monolithic. Do not spawn ODR-0008a/b/c. The arguably-real axis here is **attachment-class** (Property vs LegalEstate), which is Kendall's separate four-way alternative held-as-live (S008 Q2 / ¬ßQ2a) ‚Äî *not* the UFO axis ‚Äî and even that is already expressed losslessly by `rdfs:domain` with no module needed. Sharpened re-open trigger, inheriting ODR-0024 R4's live trigger which R2 explicitly inherits: **spawn ODR-0008a/b/c only when (i) a named consumer query is tabled whose answer needs an entailment true on one UFO axis and false on another (turning that axis off changes the answer), AND (ii) the leaves that query ranges over carry an actual gUFO `rdf:type` partition in the corpus ‚Äî not a bearer-domain partition.** Both conjuncts, or no spawn. Absent (ii) the split has no input; absent (i) it has no consumer.

## Q5 ‚Äî Preserve the insight? **REVISE ‚Äî gUFO `rdf:type` annotations, gated; not modules, not SHACL**

If the WG wants the Quality/Mode insight recorded, the cheapest honest vehicle is per-leaf gUFO `rdf:type` annotations *only on leaves where the type is uncontested* (`gufo:Quality` on the EPC/council-tax Quale-in-Region leaves), explicitly **omitting** the straddlers (`priceQualifier`) and re-sorters (`ownershipType`) until adjudicated ‚Äî annotation does not force a namespace and is reversible. Do **not** reach for SHACL: a shape implies a validation consumer that does not exist, and minting node-shapes per axis re-incurs the structure cost I am rejecting. `dct:subject` is acceptable as a lighter alternative if even gUFO typing is contested. Annotations make the meta-category *recoverable by query* without committing a single permanent IRI ‚Äî which is the whole point of preferring annotation to a module: it discharges the insight at the lowest cost and reverses for free if the typing turns out wrong.

## Withdrawal conditions (per contested question ‚Äî what makes me concede)

- **Q1 (load-bearing):** I concede the moment a panellist tables a **specific, reviewable consumer query** (SPARQL text, named consumer) whose correct answer-set changes when the Quality/Mode (or legal-estate) axis is turned off ‚Äî i.e. an entailment true on one axis and false on the others. One real query meeting Allemang's "turning the axis off changes an answer" test turns Q1 to AFFIRM.
- **Q2 (partition key):** I concede if the panel produces a **rigid adjudication rule** that assigns `priceQualifier` and `ownershipType` to exactly one axis each, stably, such that no emitted leaf straddles or re-sorts ‚Äî a partition with no boundary-crossing members is a valid module key.
- **Q3 (module reality):** For **a/b** ‚Äî I concede they are real modules once a **per-leaf gUFO typing pass over the 69 Property leaves is actually run and committed to the corpus** (not merely asserted), so the namespace would carry computed content rather than precede it. For **c** ‚Äî I concede only if someone shows an entailment ODR-0008c would license that `rdfs:domain opda:LegalEstate` does not already deliver; until then it stays a one-line query, not a module.
- **Q4 (disposition):** I withdraw the REJECT and move to SPAWN if **both** conjuncts of my sharpened trigger are satisfied at once ‚Äî a named axis-discriminating query AND an actual gUFO `rdf:type` partition over its leaves. I will also concede that *attachment-class* (Property vs LegalEstate) modularization is worth tabling if a consumer needs the two leaf-sets dereferenceable at distinct namespaces ‚Äî but that is Kendall's four-way, not the UFO split, and I would still ask why `rdfs:domain` does not already suffice.
- **Q5:** I concede that SHACL (over annotations) is the right vehicle the moment a **named validator** needs to enforce the Quality/Mode distinction at a data boundary; absent that consumer, annotation is the correct floor and I hold to it.

**Net:** REJECT the UFO-axis spawn; keep ODR-0008 monolithic; preserve the insight as gated gUFO annotations. The trigger fired for *convening*; the substantive ¬ßQ2a(a) condition ‚Äî a counted UFO-typed leaf-set plus a named load-bearing query per axis ‚Äî remains exactly as unmet as it was in Session 025. The walk gave us a bearer partition, not the UFO partition this question needs.
