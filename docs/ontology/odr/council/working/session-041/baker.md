# Baker — session-041 working position

**Voice:** Tom Baker (DCMI; co-author, *SKOS Reference* W3C Rec 2009; co-author, *DCMI Abstract Model* 2007).
**Lens:** metadata-vocabulary governance; the DCMI value-string-vs-non-literal-value distinction; Vocabulary Encoding Schemes; stewardship/maintenance cost of minting terms.

My canonical test, stated once and applied throughout: in DCMI terms a controlled term is rendered as a **non-literal value identified by a URI** (a member of a *Vocabulary Encoding Scheme*) *when it is a governed vocabulary with identity, definitions, and mappings*; it is rendered as a **value string** *when it is a transient or purely-documentary label* (DCMI Abstract Model, Powell, Nilsson, Naeve, Johnston & Baker 2007, §§"Value strings" and "Vocabulary encoding schemes"; Baker, Bechhofer, Isaac, Miles, Schreiber & Summers, "Key Choices in the Design of SKOS," *J. Web Semantics* 20 (2013), §"Concepts as resources, not as terms"). The whole of my ballot is that test applied to two different value-spaces that have been forced onto one predicate.

---

## Q1 — String literal vs resource (IRI)

**Baker:** **REVISE. Ballot: FOR the proposition, but split by axis.** The proposition is right that *a* controlled term should be a resource, but it is framed as if `opda:ufoCategory` were one vocabulary. It is two, and DCMI's value-string-vs-VES test resolves them to *opposite* answers — so a single AFFIRM or a single REJECT both mislabel half the data.

Apply the test (DCMI Abstract Model 2007, §"Vocabulary encoding schemes"; "Key Choices" 2013, §2):

- **The 9-term UFO class axis** (`Substance Kind`, `Relator`, `RoleMixin`, `Event`, … — BRIEF fact 2, class axis) *passes* the VES test on identity and mappings: the nine categories have stable identity (they are Guizzardi's, with definitions), and they have an obvious external alignment target — `gufo:Kind`, `gufo:Relator`, `gufo:RoleMixin`, `gufo:Event` (BRIEF fact 8). A value with a definition and an exact external counterpart is, by my test, a **non-literal value**: it should be a `skos:Concept` in a scheme (`opda:UFOCategoryScheme`), carrying `skos:definition` and a `skos:closeMatch` to its gUFO IRI. This is exactly the choice ODR-0011 §Option A already made for *every other* enum in the corpus ("each enum becomes a `skos:ConceptScheme` … giving each value a dereferenceable URI and a governable label"); leaving the UFO axis as bare strings is an unexplained exception to the corpus's own settled rule.
- **The "Quale-in-Region" scheme axis** (BRIEF fact 2, scheme axis; 35 of 47 tags) *fails* the VES test, but for a reason the proposition does not see. ODR-0030 Rule 2 already ruled these are **register-deference** — the value space is closed by EPC/HMLR/INSPIRE, *not* by a UFO judgement. A label that records "this scheme's values are governed elsewhere" is a **purely-documentary value string** in my sense: it has no independent identity to mint, and minting `opda:QualeInRegionCategory` as a governed concept would manufacture an identity DCMI says should not exist (DCMI Usage Board practice: *do not mint a term for a label that is not itself a maintained concept* — DCMI Usage Board, "Process and Principles," and the same discipline I applied at ODR-0011 §1a "one named steward per scheme"). For this axis the **string is correct** — but the predicate carrying it should not be the same predicate that carries the class axis (see Q2).

So my verdict is neither the proposition's blanket AFFIRM nor Cagle's blanket REJECT: **resource for the class axis, string for the scheme axis, and the two must stop sharing a predicate.** A single `opda:ufoCategory` doing double duty over a minted-VES value-space and a documentary value-string-space is, in DCMI terms, a **governance defect** — it is a property with two incompatible value-spaces and therefore no statable range discipline. (The artefact already shows the symptom: `opda-vocabularies.ttl` emits `"Substance Kind label"` and `"Role label"` where ODR-0011 §8a's own generator-emission spec wrote `"SubstanceKindLabel"` and `"RoleLabel"` — the value-space has already drifted from its own register because nothing governs a free `xsd:string`. Typo-safety is not a hypothetical here; the drift has happened.)

**Cross-talk — Allemang.** Dean, your "model the minimum that lets the data do work" (*Semantic Web for the Working Ontologist*, 3rd ed. 2020, Ch. 1) is exactly why I split rather than affirm-all. The minimum that lets the *class-axis* data do work is a resource — because the work it must do is align to gUFO and group the `/ontology/category` pages without prose-parsing, and a string does neither. But the minimum that lets the *scheme-axis* data do work is a string, because its work is already done by the external register's `sh:in`; minting a UFO concept there is model *above* the minimum — the speculative configurability your rule (and Karpathy's "no abstractions for single-use code") forbids. We agree on the principle; I am applying it twice with two answers.

**Cross-talk — Gandon.** Fabien, at ODR-0011 session-011 Q5 you pressed the lifecycle/typing discipline and I conceded the three-case mechanism. The same discipline indicts the present state: a `skos:Concept` can carry `owl:deprecated` + `dct:isReplacedBy` and survive a regulatory retention window (ODR-0011 §5a, the Pandit amendment we both signed); a bare `xsd:string` literal `"Substance Kind"` cannot be deprecated, cannot be replaced, cannot be dated. If the UFO axis is governed metadata — and ODR-0011 §8a says it is, with dual `dct:source` — then it needs the lifecycle affordances only a resource has. I am refining your typing concern into an identity concern: the class axis fails *governability*, not just typing, as a literal.

## Q2 — Markup & graph placement

**Baker:** **REVISE. Ballot: FOR, with a precise amendment on (a), (b), (c).**

**(a) Predicate typing.** Split-by-axis again:
- For the **class axis as a resource**, the predicate becomes an `owl:ObjectProperty` (`opda:Class → opda:Concept-in-UFOCategoryScheme`). This is forced by Q1: a property whose value is a resource is an object property. But — and this matters for the quarantine — typing it `owl:ObjectProperty` does *not* make it axiomatic; in DCMI/SKOS practice the alignment edge `skos:closeMatch` to gUFO is itself defined as a *mapping* relation that asserts no class-subsumption (SKOS Reference 2009, §10 "Mapping Properties": `skos:closeMatch`/`exactMatch` relate concepts "with a view to" mapping and are explicitly *not* logical equivalence; the SKOS primer warns `exactMatch` is not `owl:sameAs`). So the resource representation stays inert by *construction*, not merely by graph placement.
- For the **scheme axis as a string**, if it survives at all (I argue below it should be retired/renamed), `owl:AnnotationProperty` is the honest typing — it is documentary by stipulation (ODR-0030 Rule 7's "documentary annotation, no logical axioms"). `owl:DatatypeProperty` over-promises: a datatype property is part of the logical signature even when domain-less, which is precisely the false signal Knublauch's standing condition (BRIEF fact 4) exists to prevent.

**(b) Graph placement — this is the load-bearing finding, and I am unequivocal: the current placement is indefensible and must move to `opda-annotations.ttl`.** BRIEF fact 3 and the corpus agree: the declaration sits at `opda-classes.ttl:90` and *every* per-term tag is in a non-annotation graph; `opda-annotations.ttl` carries zero `ufoCategory` triples. ODR-0030 Rule 1 says in terms that these "remain **annotation-graph-only** … retention lapses if it is breached." **The breach already exists in the committed artefact.** This is not a future risk to weigh — it is Cagle's re-open trigger (i) ("ever made reasoned-over … moved out of the annotation-graph quarantine into a graph that drives inference or SHACL," BRIEF fact 4) **already fired**, because the triples are physically in the reasoned union (BRIEF fact 5: "still **physically in the reasoned graph**, which is what Rule 1 / trigger (i) speak to"). From the DCMI Abstract Model's standpoint this is a description-set-boundary error: advisory provenance metadata is asserted in the graph reserved for the resource's *defining* axioms. The fix is mechanical and mandatory: emit the declaration *and* all per-term tags into `opda-annotations.ttl`. Doing so is the precondition for *any* of the enrichment in Q1 — you cannot promote the class axis to a gUFO-aligned VES while it sits in the wrong graph, because that compounds the breach.

**(c) Governed SKOS scheme + SHACL `sh:in`.** Yes to the scheme; **no to a SHACL `sh:in` on it.** A `skos:in`-style closure on the *class axis* (constraining a class's `ufoCategory` to the nine concepts) is a constraint in the reasoned/validated layer — which re-imports the exact "drives … SHACL" half of trigger (i) the move in (b) is meant to cure. DCMI's integrity discipline for a VES is internal to the scheme (one primary scheme per concept; unique prefLabel per language — SKOS Reference §S14, the constraints I co-authored), *not* an `sh:in` gate on the annotating predicate. Govern the **value-space** (the scheme is closed, its nine members fixed, each with a gUFO `closeMatch`); do **not** SHACL-police the **annotation**. That keeps the resource representation strictly outside the validated graph and honours Knublauch's condition (BRIEF fact 4).

**Cross-talk — Gandon.** Fabien, on predicate typing & dereferenceability you are the right person to hold me to account: I am arguing the value becomes a resource (object property) *while* insisting nothing logical follows. The reconciliation is the SKOS mapping-property semantics above — `closeMatch` dereferences, aligns, and is queryable, yet asserts no subsumption. If you think object-property typing inevitably leaks into the signature in a way `closeMatch` cannot fence off, that is the one place I want your rebuttal, because it is the seam of my Q1+Q2 position.

## Q3 — Web pages / documentation IA

**Baker:** **AFFIRM (with one honesty constraint). Ballot: FOR.** The dereferenceable-resource case I made in Q1 *is* the web-pages case: a `skos:Concept` with a stable `/pdtf` URI is, by DCMI design, a thing that resolves to a page (DCMI Abstract Model 2007, §"Resources and their descriptions"; the "Cool URIs for the Semantic Web" / *Linked Data* principle that every minted term dereferences). So:

- **Per-category pages + a category index** — yes, and they fall out for free once the class axis is a scheme: each of the nine concepts is a page, the scheme is the index. This is the standard SKOS-publication pattern (the pattern Skosmos exists to render, which this repo already configures).
- **`/pdtf/ufoCategory` as a dereferenceable term page** — yes; the *predicate* dereferences today (BRIEF fact 1) and should keep doing so, now correctly typed per Q2.
- **Per-term UFO badges** — yes, but the badge must link to the concept page, not display a bare string; a badge that is just rendered prose re-introduces the prose-parsing problem the whole exercise removes.
- **The single `/ontology/foundational-ontology` hub** — keep it as the narrative/honesty home (it already carries the ODR-0030 verdict). The scheme pages are the *structured* view; the hub is the *prose* view. Two artefacts, one per register — the same two-artefact discipline I helped set as default at ODR-0011 session-011.

**The honesty constraint (non-negotiable, and it is mine to press as the metadata-governance voice):** every category-concept's page MUST carry the `skos:scopeNote` ODR-0030 Rule 7(b) mandates — (i) the categories are UFO's; (ii) the quality categories descend from DOLCE's Quality/Quale/Region (Masolo et al., WonderWeb D18, 2003); (iii) the majority of tagged properties fall under that DOLCE-derived apparatus — and the page MUST render "UFO-**informed**," never "UFO-grounded." A controlled vocabulary that hides its own provenance and centre-of-gravity has failed the one governance service DCMI says a scheme exists to render (this is Guarino's swing condition from session-040, and I concur in full as a *metadata* obligation, not merely a courtesy). The honesty disposition is *better served* by the resource representation, not threatened by it: a `scopeNote` is a first-class property of a `skos:Concept`/scheme and travels with the term; bolted onto a free-text string it does not.

**Cross-talk — Allemang.** Dean, the risk you would rightly raise is that per-category pages *entrench* a layer session-040 called "optional, droppable" — Cagle's whole DA thesis. My answer: dereferenceable ≠ entrenched. A SKOS scheme is the *most reversible* possible representation — retiring it is `owl:deprecated true` + a `skos:historyNote` on nine concepts (ODR-0011 §5a), leaving every old reference resolvable for its retention window. A scheme is easier to retire cleanly than 47 scattered string literals, which have no deprecation affordance at all. Pages do not raise the cost of Option D; the scheme *lowers* it.

## Q4 — What else to mark up

**Baker:** **REVISE. Ballot: FOR, narrowly — mint almost nothing new.** My governing rule here is stewardship cost: every minted term is a maintenance liability with a named steward and a lifecycle (ODR-0011 §1a; DCMI Usage Board discipline — a vocabulary you mint is a vocabulary you must maintain, define, version, and defend, *J. Web Semantics* 2013 §"The cost of meaning"). The default for the candidate list is therefore **prose**, and the bar for promotion to a structured resource is the VES test from Q1: *governed identity + definitions + an external alignment target you will actually maintain.*

Applying it to the BRIEF Q4 candidates:

- **OntoClean meta-properties (+R/−R, +I/−I, +D/−D).** **Stay prose — do NOT mint.** These are a *decision procedure* applied at design time (ODR-0030's "the modelling process is governed by a UFO/OntoClean decision procedure; the artefact carries an inert provenance record"). A decision procedure is not a controlled vocabulary; minting `opda:rigidity "+R"` as structured metadata would assert that the *artefact* carries the judgement, when ODR-0030's honest verb is that only the *process* did. This is the one place enrichment would actively lie. Prose in the ODR/scopeNote; nothing in the graph.
- **The gUFO `rdf:type` markers (ADR-0034).** **Already correctly placed — leave exactly as is.** They are in `opda-descriptive-annotations.ttl` (verified: `gufo:` is bound *only* there), quarantined per ADR-0034 and Knublauch's condition. Do not promote, do not move, do not extend. Note they are the *one* place the proposition's "`skos:exactMatch` to gUFO" idea already half-exists — and the corpus has **zero** `skos:exactMatch`/`closeMatch` to any `gufo:` IRI today (verified). So the Q1 class-axis alignment I propose is net-*new* minting of mapping edges; it must be weighed as new stewardship, and I accept it only because nine `closeMatch` edges to a published, stable gUFO are low-maintenance and high-payoff (queryability + alignment). Anything beyond those nine fails the cost test.
- **The Relator/RoleMixin "founds" edge** (ODR-0030's "strictly irreducible thing"). **Stays prose for now; it is the one candidate I would *consider* later** — but only if a *driver* appears (a query or a consumer that needs to traverse "Relator founds which RoleMixins"), per the generator-first discipline (ODR-0011 §Purpose-scheme, "emission gated on its first driver"). Until then, minting `opda:founds` is speculative configurability.
- **UFO-L Hohfeldian Claim-Right↔Duty correlativity.** **Stays prose / stays UFO-L-shaped where it is** (ODR-0030 Rule 6). This is the deontic core, uncontested; it is not a *category-axis* metadata question and does not belong in the `ufoCategory` scheme. No new markup from this council.
- **The IAO crosswalk (ODR-0030 Rule 4, adopt-now).** **Mint — but this is already decided, and it is the *model for how to do Q1 right*.** ODR-0030 Rule 4 specifies a thin, *referenced-not-imported* set of `skos:exactMatch`/`closeMatch`/`rdfs:subClassOf` edges to `obo:IAO`, in the **annotation graph**, on the `prov:Entity` precedent. That is precisely the discipline I want for the UFO class axis: external, referenced, mapping-only, annotation-graph-resident. The IAO crosswalk is the template; the UFO-category scheme should be built the same way, in the same place, never `owl:imports`.

**Net Q4:** mint the nine gUFO `closeMatch` edges (as part of the Q1 class-axis scheme, in the annotation graph); ship the IAO crosswalk per Rule 4; **mint nothing else.** Everything else on the candidate list stays prose, because the stewardship cost is real and the VES test is not met.

**Cross-talk — Gandon.** Fabien, this is where you and I should align tightly: your dereferenceability concern and my stewardship concern are the same coin. A term that dereferences is a term someone must keep dereferencing — define, version, deprecate, defend. I am holding the *mint* list to four classes of edge (nine UFO `closeMatch`, the IAO crosswalk) for exactly the reason you would not want dangling URIs: an under-maintained mapping is worse than honest prose. If you want *more* of the Q4 list marked up for query-reach, the burden is to name the maintainer and the driver — DCMI will not mint on spec.

|---|---|---|---|
| Q1 string-vs-resource | **REVISE** | **FOR** | Resource (SKOS concept + gUFO `closeMatch`) for the 9-term **class axis**; **string** for the register-deference **scheme axis**; the two must stop sharing one predicate (double-duty = governance defect). |
| Q2 markup & placement | **REVISE** | **FOR** | (a) `owl:ObjectProperty` for the resource axis / `owl:AnnotationProperty` for any surviving string; (b) **move declaration + all tags to `opda-annotations.ttl` — the quarantine breach already exists and trigger (i) has fired**; (c) govern the scheme's value-space, do **NOT** add a SHACL `sh:in` (that re-breaches the quarantine). |

| Q4 what else | **REVISE** | **FOR (narrowly)** | Mint only the nine gUFO `closeMatch` edges + the IAO crosswalk (Rule 4 is the template), both annotation-graph / referenced-not-imported. OntoClean meta-properties, the "founds" edge, UFO-L, gUFO types: **stay prose / stay put.** Default is prose; the VES test (governed identity + definitions + a maintained alignment target) is the only gate to a resource. |

**Overall:** **FOR the enrichment proposition, as revised** — but the enrichment is *conditional on first curing the placement breach* (Q2b). Promote the class axis to a gUFO-aligned SKOS VES and give it pages; keep the register-deference axis as documentary strings under a *different* predicate; mint nothing else. The single largest defect in the status quo is not that the value is a string — it is that one predicate, sitting in the reasoned graph, carries two incompatible value-spaces. Fix the placement and split the predicate, and the rest of the proposition follows.
