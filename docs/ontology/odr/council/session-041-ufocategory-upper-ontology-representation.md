## Context

ADR-0044 Phase 5c (2026-06-15) promoted the UFO meta-category from documentary `scopeNote` free-text to a structured `opda:ufoCategory` facet — declared an `owl:DatatypeProperty (range xsd:string)`, dereferencing at `/pdtf/ufoCategory`, asserted on 39/40 classes + 47 schemes. The operator asked the council four questions: **Q1** should the value be a string literal or a resource (IRI)?; **Q2** how should it be marked up (predicate typing + graph placement)?; **Q3** how should the web pages / documentation present `ufoCategory` and the upper-ontology layer?; **Q4** what *other* upper-ontology / modelling-framework metadata should be marked up?

The proposition under review (the **enrichment** framing): represent `ufoCategory` and the upper-ontology layer as structured, dereferenceable resources — mint a governed SKOS category vocabulary with gUFO alignment, correct the markup/placement, present first-class web pages, and extend structured markup to other upper-ontology metadata.

**The pre-flight scope check surfaced a load-bearing tension the question did not name.** Phase 5c shipped *one day after* session-040 / ODR-0030 (2026-06-14), whose **Rule 1** ruled that `opda:ufoCategory` tags "remain **annotation-graph-only** … the ODR-0029 quarantine is load-bearing — **retention lapses if it is breached**," and whose DA (Cagle) holds a live re-open trigger (i): *"the `ufoCategory` layer is ever made reasoned-over."* So the design question rides on top of a **possible quarantine breach**. Three corpus facts, **independently verified by the Queen and by three panellists**:

1. **All ~87 `ufoCategory` triples — and the predicate's own declaration (`opda-classes.ttl:90`) — sit in the non-annotation (reasoned-union) graphs** (`opda-classes/agent/claim/property/descriptive/governance/transaction.ttl` + `opda-vocabularies.ttl`); `opda-annotations.ttl` carries **zero** (`grep -c → 0`). ODR-0030 Rule 1 is breached *in the shipped bytes*.
2. **The three-graph CI gate is blind to this class of breach.** `three_graph_test.py`'s five checks forbid `sh:` in annotations, `owl:imports`/advisory in shapes, and check `sh:targetClass` resolution — **none forbids an inert predicate in the classes graph.** The breach ships green; Knublauch's session-040 standing condition is, for this predicate, unenforced.
3. **The ODR-0030 Rule 7(b) DOLCE disclosure never shipped** on the predicate: `opda:ufoCategory`'s `scopeNote` (`opda-classes.ttl:96`) carries only "promotes … to a structured, queryable facet," not the three-part disclosure (the categories are UFO's; UFO's quality categories descend from DOLCE's Quality/Quale/Region — Masolo et al., WonderWeb D18, 2003; the majority of tagged properties fall under that DOLCE-derived apparatus). Per-class fragments exist; the consolidated predicate-level disclosure does not.

A secondary defect: the predicate does **double duty** over two incompatible value-spaces — the 9-term UFO endurant/perdurant axis on classes (`Substance Kind`, `Relator`, `Role`, …) and a *different* scheme axis (35× `"Quale-in-Region"`, plus `"Substance Kind label"`, `"Phase label"`, …) already ruled register-deference by ODR-0030 Rule 2.

## Question 1 — String literal vs resource (IRI)?

**4–2–0 FOR resource (REVISE). The value should be an IRI; the live split is *mint-local-SKOS* vs *reference-gUFO-directly*; the register-deference scheme axis is split off. Cagle (DA) HELD for a governed string.**

**Guizzardi:** REVISE / FOR. "A bare `"Substance Kind"` string discards [the category's identity + axiomatic profile] and has no governance surface." Mint a *local* SKOS concept in `opda:UFOCategoryScheme` with `skos:exactMatch`/`closeMatch` to the gUFO IRI — *not* the gUFO IRI reused as a data subject — citing the SKOS Reference §10 mapping idiom and his own gUFO OWL-2-DL undecidability result (Almeida, Guizzardi, Sales & Fonseca) as the warrant that the alignment must be a non-reasoned pointer.

**Guarino:** REVISE / FOR. "A UFO/OntoClean category is not a string; it is a type qualified by meta-properties … the faithful representation of an entity-with-identity is an IRI" (Guarino & Welty, "An Overview of OntoClean," *Handbook on Ontologies* 2nd ed. 2009, §3–4). But — load-bearing refinement of Guizzardi — *never* a direct `gufo:Kind` reference or object-level `rdf:type`: "gUFO's IRIs carry gUFO's OWL axioms; a direct binding in a reasoned graph re-imports the modal/mereological commitments … not faithfully expressible in OWL 2 DL — exactly what ODR-0030 Rule 3 forbids." The SKOS-concept-with-`exactMatch` is "the indirection that gets gUFO alignment *and* respects the level boundary" (Guarino, "The Ontological Level," 1994). Parts from Cagle: "Inertness is achieved by quarantine (Q2), not by degrading the referent to a string."

**Baker:** REVISE / FOR — applying the DCMI value-string-vs-non-literal-value test (DCMI Abstract Model, Powell, Nilsson, Naeve, Johnston & Baker 2007; Baker et al., "Key Choices in the Design of SKOS," *J. Web Semantics* 20, 2013) twice, with two answers: "The defect is not 'string vs resource' — it is **one predicate carrying two incompatible value-spaces**. Split it." The 9-term class axis passes the Vocabulary-Encoding-Scheme test (governed identity + definitions + a gUFO alignment target) → SKOS concept with `skos:closeMatch` to gUFO; the 35× "Quale-in-Region" axis is register-deference (ODR-0030 Rule 2) → "a purely-documentary value string; minting a UFO concept there manufactures identity DCMI says not to." Verified two corpus facts: ungoverned strings **already drifted** (`"Substance Kind label"` vs ODR-0011 §8a's own spec `SubstanceKindLabel`); **zero** `skos:exactMatch`/`closeMatch` to any `gufo:` IRI exists today (the edges are net-new).

**Gandon:** REVISE / FOR. A resource value is what makes the term first-class on the web (Linked Data principles; RDF 1.1 Concepts). Refined Baker's "literal→resource" into a **two-scheme split** (genuine UFO-category concepts vs the 35 register-deference VES tags).

**Allemang (Queen):** REVISE / **AGAINST the framing**. "The framing's central move — *enrich → mint a local concept scheme* — is the textbook over-modelling error my own book names" (*SWWO* 3rd ed., Ch. 12–13). For the **class axis**, the reuse-before-you-mint rule forces a **direct gUFO IRI reference** (`gufo:Kind`, `gufo:Relator`): "Minting `opda:SubstanceKindCategory` and then attaching `skos:exactMatch gufo:Kind` is building your own copy of a thing that exists so you can declare it identical to the thing that exists … the long way round to `gufo:Kind`." For the **scheme axis**, split it off to a string / separate predicate (ODR-0030 Rule 2 already conceded these are "not UFO doing categorial work"). "Neither half is 'a governed local SKOS scheme,' which is why I cannot AFFIRM."

**Cagle (DA):** REVISE / AGAINST. "A discriminating label with no managed semantic relations is a SHACL-validated datum, not a SKOS concept" — SKOS Primer (W3C Note 2009-08-18) §1.3: mint a concept scheme for *managed semantic relations*, "not merely to hang a definition on a label. Nine flat, non-hierarchical category names with no inter-concept relations is the textbook case the Primer warns is *over*-modelling." Floor: governed `xsd:string` + a SHACL `sh:in` enum (the in-repo idiom — `opda:ownerType`/`opda:evidenceType` via `sh:targetSubjectsOf`, ODR-0024 R5/R6, verified). On the `gufo:`-IRI-value option he rebuts Guizzardi: "you cannot vote No on deepening at Q2 [session-040] and Yes on gUFO-IRI-valued tags at Q1; they are the same act at different granularity." **Disposition: HELD.** Withdrawal condition: "a **second** consumer … that needs `skos:broader`/`skos:mapping` edges between the categories themselves" — unmet → **held-as-live**.

**Vote Q1: 4–2–0 FOR resource (REVISE).** Near-unanimous (5–1) that the value should be an *IRI* (only Cagle holds for a string); the ballot splits 4–2 on the proposition-as-framed (local SKOS scheme) because Allemang holds for *direct gUFO reference* over local minting. The decisive counter to the reuse argument (Guarino + Baker): only a *local* node can bear OPDA's own `skos:definition` (the OntoClean meta-property signature), the DOLCE disclosure, and the honesty framing — a foreign `gufo:` IRI cannot be annotated by OPDA — and **`skos:closeMatch` (not `exactMatch`)** defuses the "long way round to identity" objection, since OPDA asserts *alignment*, not *identity*.

## Question 2 — Markup & graph placement (the load-bearing question)

**5–1–0 (Cagle AGAINST-as-framed → WITHDRAWN on the adopted landing): retype `owl:AnnotationProperty`, relocate all tags + declaration to `opda-annotations.ttl`, add a 6th CI gate. This is the precondition for Q1/Q3/Q4.**

**Gandon:** REVISE / FOR — the decisive resolution. ODR-0030 calls `ufoCategory` "inert, non-axiomatic," yet it is typed `owl:DatatypeProperty` — "a logical property in the OWL signature." The conformant fix is **`owl:AnnotationProperty`** (OWL 2 Structural Specification §10.1 / §5.5 — annotation assertions "have no effect on the logical meaning of an ontology"): "That makes inertness *intrinsic to the term* … and makes Cagle's re-open trigger (i) un-fireable by accident — an annotation property cannot be 'made reasoned-over' without an explicit retyping. **An IRI value stays `AnnotationProperty`, NOT `ObjectProperty`.**" Reads the breach precisely: "a *placement* breach … not an *inertness* breach (0 entailed triples, nothing drives SHACL)" — fixed by relocation + retype as one move; the ODR-0004 §3a gate "checks `sh:` in the annotation graph but NOT the converse."

**Allemang (Queen):** REVISE / FOR (placement). "The `owl:DatatypeProperty` is a *category error against the authors' own stated intent*" (the predicate's `rdfs:comment`: "records the foundational-ontology commitment without entailing logical axioms"). "**Move every `ufoCategory` triple … into `opda-annotations.ttl`** … or ODR-0030's AFFIRM lapses by its own terms … this is **Cagle's re-open trigger (i) already fired in the committed corpus**." REJECT `sh:in` in the main union: "a `sh:in` shape constraining `ufoCategory`'s values would put the predicate *into SHACL* — that is re-open trigger (i) a second time."

**Guizzardi:** REVISE / FOR. "Promoting the value to a resource WITHOUT relocating fires Cagle's re-open trigger (i) verbatim. Promoting WITH relocation retires a latent fault." Extend the CI ASK gate to catch the converse leak. `sh:in` = "typo-guard only, never a correctness reasoner."

**Guarino:** REVISE / FOR. "A meta-level annotation sitting in the object-level graph is a category error in the literal sense — the level boundary that the whole OntoClean apparatus rests on is, right now, not drawn in the artefact." `owl:AnnotationProperty` is "inert *by its own definition*, under any regime, which is what Knublauch's standing condition actually needs" — stronger than "the contingent shallowness of the current regime." `sh:in` permitted *only* as a closed-vocabulary editorial guard on the predicate's own value, "never as a lever on object-level conformance" (consistent with ODR-0027 R5 + ODR-0004 §3a).

**Baker:** REVISE / FOR. "The quarantine breach **already exists in the committed artefact** … the precondition for any enrichment." Typing: ObjectProperty for a resource axis / AnnotationProperty for any surviving string — *but flagged the "object-property-vs-inertness seam" as the place he wanted Gandon's rebuttal* (which Gandon supplied: AnnotationProperty even with an IRI value). Do **not** add `sh:in` to the main shapes graph (re-breaches the quarantine).

**Cagle (DA):** REVISE / AGAINST-as-framed. "The current placement … is the literal text of ODR-0030 Rule 1's prohibition and the literal text of my re-open trigger (i). **Trigger (i) has already fired.**" Move declaration + all tags to `opda-annotations.ttl`; **add a sixth check** to `three_graph_test.py`: `ASK { GRAPH opda:classes { ?s opda:ufoCategory ?o } } → FALSE` — "so Knublauch's standing condition is actually *enforced* and not merely *asserted*." `sh:in` YES, minted scheme NO. **Disposition: WITHDRAWN** — withdrawal condition stated verbatim: "(i) predicate retyped `owl:AnnotationProperty`; (ii) declaration + all tags relocated to `opda-annotations.ttl`; (iii) a CI `ASK` gate forbidding `ufoCategory` in the classes graph" — **all three are the adopted landing**, so the HOLD lifts.

**Vote Q2: 5–1–0 as-framed → 6–0 on the amended landing.** Unanimous on the two core moves (retype `owl:AnnotationProperty` [IRI-valued — Gandon's inertness argument beats both `DatatypeProperty` and `ObjectProperty`]; relocate all ~87 tags + declaration to `opda-annotations.ttl`) + the 6th CI gate. `sh:in` lands as a **tag-only editorial guard** (validate the tag value ∈ the nine concepts) kept *out* of the main instance-validation union — honouring Cagle/Guarino's typo-governance without re-firing trigger (i) (Allemang/Baker). Cagle's withdrawal condition is met; **the DA withdraws.**

## Synthesis (Queen — Allemang)

The council **affirms the spirit of the enrichment proposition and rejects its mechanism**: enrich `opda:ufoCategory` by **reference and disclosure under a restored quarantine** — not by minting into the reasoned graph. The proposition was right that the value should usually be a resource (5–1) and right about web pages (5–0–1); it was wrong to leave the tags where Phase 5c put them, and the session's load-bearing finding is a **shipped breach of the binding prior**, not an aesthetic preference.

**The breach is the centre of gravity.** Three independent verifications (Queen + Guizzardi + Gandon + Cagle, each at file:line) confirm that all ~87 `ufoCategory` triples and the predicate declaration sit in the reasoned-union graphs while `opda-annotations.ttl` carries zero — *exactly* ODR-0030 Rule 1's prohibition and Cagle's re-open trigger (i), shipped green because the three-graph gate has no check for it. Gandon drew the distinction that lets the council act precisely: this is a **placement** breach, not an **inertness** breach (nothing is entailed today under ODR-0029's shallow regime), so the cure is mechanical and cheap. **The cure is the precondition for everything else**, and it is what keeps Cagle's session-040 Option-D revert dormant: the council adopts enrichment *only bundled with, and downstream of,* the quarantine restoration the DA demanded. That is the thread that holds the whole verdict together — every "FOR" below is conditional on Q2 landing first.

**Q1 — resource, by local SKOS concept `closeMatch`-aligned to gUFO, with the predicate split.** My own ballot was AGAINST local minting (reuse `gufo:` directly), and I record that my position lost on the merits: Guarino and Baker carried the decisive point that *only a local node can bear OPDA's own `skos:definition` (the OntoClean signature), the DOLCE disclosure, and the honesty framing — a foreign `gufo:` IRI cannot be annotated by OPDA* — and `skos:closeMatch` (not `exactMatch`) answers my "long way round to identity" objection by asserting alignment rather than identity, which is also the honest claim (Guizzardi's own OWL-2-DL result forbids treating them as logically identical). Baker's verified typo-drift (`"Substance Kind label"` vs the spec's `SubstanceKindLabel`) is the empirical proof that the ungoverned-string status quo has *already* failed governance. The **predicate splits**: the 9-term UFO class axis takes the `closeMatch`-aligned concept; the 47 register-deference scheme tags (ODR-0030 Rule 2) demote to a documentary string / separate predicate and stop presenting as "UFO doing categorial work." Cagle's "nine flat labels = enum, not a scheme" holds as live dissent — its re-open trigger (a second consumer needing inter-category `skos:broader`/mapping edges as data) is noted; Guizzardi's observation that the UFO categories *do* form a latent taxonomy (endurant/perdurant, sortal/non-sortal) is the standing answer if that consumer materialises.

**Q2 — the mechanical cure.** Retype `opda:ufoCategory` `owl:AnnotationProperty` (IRI-valued — inertness intrinsic, OWL 2 §10.1/§5.5; Gandon's argument that an IRI value stays an annotation property, *not* an object property, settles Baker's seam); relocate the declaration + all ~87 tags into `opda-annotations.ttl`; add the **sixth three-graph gate** `ASK { GRAPH opda:classes { ?s opda:ufoCategory ?o } } → FALSE` (closing the verified blind spot). `sh:in` is admitted *only* as a tag-level editorial guard on the predicate's own value, kept out of the main instance-validation union — the synthesis that gives Cagle/Guarino their typo-governance without re-firing trigger (i) into SHACL (Allemang/Baker). Unanimous on the landing; the DA withdraws.

**Q3 — pages as honest projections.** Keep `/pdtf/ufoCategory`, the per-category pages, and the badges as read-side SPARQL views that render off either representation (so they cannot entrench), and **ship the Rule-7(b) DOLCE disclosure that session-040 still owes** — Guarino's WonderWeb-D18 lineage, on the predicate `scopeNote` and the term page. Every upper-ontology surface leads with "UFO-informed, not UFO-grounded"; badges say "classified-under," never "is-a." Good IA *discloses* the optional layer; it does not deepen it (Baker: a scheme is more reversible than scattered strings).
